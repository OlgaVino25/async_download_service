# Микросервис для скачивания файлов

Микросервис помогает работе основного сайта (CMS) и обслуживает запросы на скачивание архивов с файлами.  
Создание архива происходит **на лету**: архив не сохраняется на диске, а по мере упаковки сразу отправляется пользователю.  
Архив защищён от неавторизованного доступа с помощью хеша в URL (имя папки с файлами).

## Структура каталога с фотографиями

Файлы должны быть организованы в папке `photos` (или другой, указанной через настройки):

```
- photos/
    - хеш_папки_1/
      - 1.jpg
      - 2.jpg
      - 3.jpg
    - хеш_папки_2/
      - 1.jpg
      - 2.jpg
```

Хеш – это любая строка, которая служит секретным ключом. Пример ссылки: `http://host.ru/archive/хеш_папки_1/`

## Установка и запуск

### Требования

- Python 3.7 или выше
- Установленная программа `zip` (в Linux/macOS обычно предустановлена, в Windows – через WSL)
- Docker и docker-compose (опционально)

### Локальный запуск без Docker

```bash
pip install -r requirements.txt
python server.py
```

Сервер запустится на `http://0.0.0.0:8080`.
Для проверки откройте в браузере http://127.0.0.1:8080/ – должна открыться стартовая страница.

### Запуск через Docker

1. Соберите образ:

```bash
docker build -t archive-service .
```

2. Запустите контейнер, смонтировав папку с фотографиями:

```bash
docker run -d -p 8080:8080 -v $(pwd)/photos:/app/photos archive-service
```

3. Для разработки используйте `docker-compose`:

```bash
docker-compose up --build
```

### Запуск в продакшен-режиме

Используйте docker-compose.prod.yml и deploy.sh (см. файлы в репозитории).

## Настройка

Микросервис поддерживает аргументы командной строки и переменные окружения.

**Аргументы командной строки**

| Аргумент       | Описание                                                                          | По умолчанию |
| -------------- | --------------------------------------------------------------------------------- | ------------ |
| `--photos-dir` | Путь к папке с фотографиями                                                       | `photos`     |
| `--log-level`  | Уровень логирования: `DEBUG`, `INFO`, `WARNING`, `ERROR`                          | `DEBUG`      |
| `--delay`      | Задержка (в секундах) между отправкой чанков (для тестирования обрыва соединения) | `0`          |

Пример:

```bash
python server.py --photos-dir /var/www/photos --log-level INFO --delay 0.2
```

**Переменные окружения**

Переменные окружения имеют те же имена (в верхнем регистре) и могут использоваться вместо аргументов:

- `PHOTOS_DIR`
- `LOG_LEVEL`
- `CHUNK_DELAY`

Пример:

```bash
export PHOTOS_DIR=/var/www/photos
export LOG_LEVEL=INFO
export CHUNK_DELAY=0.2
python server.py
```

_Аргументы командной строки имеют приоритет над переменными окружения._

## Как это работает

- **Асинхронный стриминг** – сервер запускает процесс zip -r - . в папке с запрошенным хешем, читает его stdout частями (чанками по 8 КБ) и сразу отправляет клиенту без сохранения архива на диск.
- **Обработка обрыва соединения** – если клиент прервал загрузку (закрыл вкладку, нажал «Отмена»), сервер перехватывает исключение ConnectionResetError, завершает процесс zip и логирует Download was interrupted.
- **Graceful shutdown** – при получении сигнала SIGINT (Ctrl+C) сервер завершает все активные zip-процессы, дожидается их остановки и только после этого выключается.
- **Настройка задержки** – параметр --delay позволяет искусственно замедлить отправку (полезно для тестирования обрыва на больших файлах).
- **Docker** – образ содержит всё необходимое, включая zip. Монтирование папки с фото позволяет обновлять файлы без пересборки образа.

## Примеры запросов

```bash
# Скачать архив с файлами из папки photos/7kna
curl -O http://127.0.0.1:8080/archive/7kna/
```

## Развёртывание на сервере

1. Скопируйте код на сервер.
2. Установите Docker и docker-compose.
3. Настройте `docker-compose.prod.yml` (укажите правильный путь к папке с фото).
4. Запустите: `docker-compose -f docker-compose.prod.yml up -d`.
5. Настройте веб-сервер (Nginx) на проксирование `/archive/` на `http://127.0.0.1:8080`.
